Capability / Requirement – Business or technical capability (e.g., SSO, Reporting, API coverage, SLA).

Current State (As-Is) – What exists today (objective, measurable).

Target State (To-Be) – What the new solution must support.

Gap Description – Explicit delta between As-Is and To-Be.

Gap Type – Functional / Technical / Process / Data / Compliance / Integration / Performance.

Business Impact – Revenue risk, operational friction, compliance exposure, etc.

Priority – Critical / High / Medium / Low (or MoSCoW).

Remediation Action – Buy / Build / Configure / Process change / De-scope.

Owner – Accountable stakeholder.

Timeline – Milestone or sprint window.
